# workout-coach — трекер силовых тренировок с движком прогрессии

## Что это
Персональное веб-приложение: журнал силовых тренировок + алгоритмический тренер
(гибрид двойной прогрессии и RIR-коррекции, мезоциклы 5+1). Пользователь один,
цель — гипертрофия. Полные требования: docs/workout-app-PRD.md,
архитектура: docs/workout-app-architecture.md.

## Constraints (нарушать нельзя)
- Vanilla JS (ES2020+), НИКАКИХ фреймворков и внешних зависимостей
- Артефакт — ОДИН файл dist/index.html, работает офлайн, mobile-first, тёмная тема
- Хранение только localStorage + JSON-бэкап; единственная точка записи — store.save()
- engine.js и analytics.js — чистые функции без DOM (тестируемость + задел v2)
- У SetLog остаются поля analysis/mediaRef (задел под видеоанализ v2) — не удалять

## Команды
- Тесты: `node test-chunk1.js` (и последующие test-chunk*.js)
- Сборка: `node src/build.js` → dist/index.html (появится в чанке сборки)

## Параметры движка (подтверждены пользователем)
- RIR-рампа по неделям мезоцикла: нед.1 → RIR 3, нед.2–3 → RIR 2, нед.4–5 → RIR 1, делоуд → RIR 4
- Перегруз: фактический RIR 0 в 2+ сетах сессии → вес не наращивать; повторно → −5%
- Делоуд: 60% рабочего веса, 50% сетов
- Прогрессия: все рабочие сеты у потолка диапазона повторов → +1 шаг веса, возврат к низу диапазона
- Калибровка нового упражнения (подтверждено 2026-07): вся первая тренировка —
  лесенка прикидок не до отказа (легко → +вес по Эпли → ... до низа диапазона
  повторов, кап ~workSets+1); рабочий вес = проекция лучшего e1RM лесенки,
  применяется со следующей тренировки. Подбор можно пропустить («знаю рабочий
  вес»). Прикидки помечаются isCalibration и исключаются из стагнации/e1RM
