# Промпт: Приложение для силовых тренировок «Встроенный тренер»

## Роль

Ты — senior frontend-разработчик и специалист по методологии гипертрофии (RIR-подход, в духе Renaissance Periodization / Mike Israetel). Создай полнофункциональное standalone веб-приложение для планирования и трекинга силовых тренировок.

## Контекст пользователя

- Мужчина, 46 лет, цель — **рост мышечной массы (гипертрофия)**
- 3 силовые тренировки в неделю по фиксированным дням (A/B/C), параллельно 3 беговые
- Пользуется приложением с телефона прямо в зале — mobile-first обязателен
- Опытный пользователь данных: хочет видеть динамику и понимать логику рекомендаций

## Технические требования

- **Один HTML-файл**: HTML + CSS + JS без сборки, открывается в любом браузере
- **Mobile-first**: крупные кнопки, ввод одной рукой, тёмная тема
- Данные — в `localStorage`
- **Обязателен экспорт/импорт JSON-бэкапа** (кнопки «Скачать бэкап» / «Восстановить из файла»)
- Никаких внешних зависимостей, требующих сети в зале (офлайн-работа)

## Структура данных

- **Упражнение**: название, мышечная группа (основная + вторичные), тип (базовое/изоляция), шаг веса (2.5 / 5 кг / и т.п.)
- **День тренировки (A/B/C)**: упорядоченный список упражнений с целевыми параметрами: диапазон повторов (напр. 8–12), число рабочих сетов, целевой RIR, время отдыха
- **Подход (лог)**: дата, упражнение, № подхода, вес, повторы, фактический RIR
- **Мезоцикл**: номер недели (1–6), флаг делоуда

## Функциональные блоки

### 1. Библиотека упражнений

- Встроенная база **~50 популярных упражнений** с привязкой к мышечным группам (грудь, спина, ноги, плечи, руки, кор): жимы, тяги, приседания, выпады, изоляция
- Пользователь **выбирает из списка** при составлении дня A/B/C (поиск + фильтр по группе)
- Возможность добавить своё упражнение и отредактировать/удалить любое

### 2. Конструктор программы

- Три дня: A, B, C — редактируемые списки упражнений
- Для каждого упражнения задаются: диапазон повторов, число сетов, целевой RIR, отдых
- Упражнения можно добавлять, убирать, менять порядок (drag или кнопки вверх/вниз)

### 3. Экран активной тренировки («тренерский режим»)

Главный экран, оптимизирован под ввод за 5 секунд между подходами:

- Выбор дня (A/B/C) → последовательный список упражнений
- Для каждого подхода показывать:
  - **Рекомендацию**: «Сегодня: 42.5 кг × 10, цель RIR 2»
  - **Прошлый результат** этого подхода на прошлой тренировке
- Быстрый ввод: вес / повторы / фактический RIR (степперы + прямой ввод)
- **Таймер отдыха**: автостарт после сохранения подхода, заданная длительность, звук/вибрация по окончании
- Возможность добавить внеплановый сет или пропустить упражнение

### 4. Движок рекомендаций (гибридная модель)

**База — двойная прогрессия:**

- Работаем в диапазоне повторов (напр. 8–12) с фиксированным весом
- Все рабочие сеты достигли верха диапазона → рекомендация +1 шаг веса, возврат к нижней границе повторов

**Корректирующий сигнал — RIR:**

- Целевой RIR по неделям мезоцикла: нед. 1 → RIR 3, нед. 2–3 → RIR 2, нед. 4–5 → RIR 1, делоуд → RIR 4–5
- Фактический RIR заметно выше целевого (легко) 2 сессии подряд → ускорить прогрессию (+вес раньше потолка повторов)
- Фактический RIR ниже целевого (слишком тяжело, 0 или отказ) → держать или снизить вес на 5–10%, не наращивать
- Логика рекомендации должна быть **объяснимой**: рядом с рекомендацией короткая причина («+2.5 кг: достигнут потолок повторов»)

### 5. Режим калибровки рабочего веса

Для упражнения **без истории** (новое или заменённое) первая сессия проходит в режиме калибровки:

- Пользователь делает разведочный подход с заведомо лёгким весом и вводит: вес × повторы × RIR
- Приложение считает e1RM по Эпли и **проецирует рабочий вес** под целевой диапазон повторов и целевой RIR (с округлением до шага веса)
- Следующий подход — проверка прогноза: если фактический RIR совпал с целевым (±1) — вес зафиксирован как рабочий; если нет — коррекция и ещё один контрольный подход
- Обычно калибровка сходится за 2–3 подхода; итог сохраняется как базовый вес для движка прогрессии
- Режим можно вызвать вручную для любого упражнения («перекалибровать») — например, после долгого перерыва
- Калибровочная сессия помечается в истории и **не участвует** в детекторе стагнации и графике e1RM как обычная точка (или помечается отдельным маркером)

### 6. Периодизация — календарные мезоциклы

- Цикл: **5 недель наращивания → 1 неделя делоуд → новый цикл**
- Приложение ведёт счётчик недель и явно показывает текущую неделю мезоцикла
- В делоуд-неделю: автоматически рекомендуется ~60% рабочего веса и половина сетов
- После делоуда новый цикл стартует с базы чуть выше предыдущей

### 7. Аналитика (отдельный экран, вне тренировки)

- **e1RM по упражнениям**: расчётный 1ПМ по формуле Эпли (вес × (1 + повторы/30)), линейный график по датам, выбор упражнения
- **Недельный объём по мышечным группам**: число рабочих сетов на группу в неделю, столбчатая диаграмма, подсветка групп вне коридора 10–20 сетов
- **Детектор стагнации**: если e1RM упражнения не растёт 3+ недели (вне делоуда) — бейдж-предупреждение и подсказка (сменить упражнение / проверить восстановление / микропериодизация)
- История тренировок: календарь-лента с возможностью открыть и отредактировать любую прошлую сессию

## UX-принципы

- Ввод результата подхода — не более 3 касаний в типовом случае
- Прошлые результаты всегда видны при вводе — без переключения экранов
- Тёмная тема по умолчанию, крупная типографика
- Никакой регистрации, никаких аккаунтов — открыл и работаешь

## Задел на v2: видеоанализ техники (НЕ реализовывать в MVP)

В MVP функция не строится, но архитектура должна оставить для неё место:

- **Модель данных**: у каждого подхода (лога) — опциональное поле `analysis: object | null` (произвольная структура: метрики, флаги техники, рекомендация) и `mediaRef: string | null`. Сериализуются в бэкап как есть
- **Интерфейс анализаторов**: движок рекомендаций принимает на вход массив «сигналов» (сейчас: повторы, RIR; в будущем: темп, амплитуда, флаги техники) — новый источник сигналов подключается без переписывания ядра
- **UI-хук**: в карточке подхода зарезервировано место под бейдж/иконку анализа (скрыто, если `analysis == null`)
- Планируемый стек v2: MediaPipe BlazePose / TensorFlow.js в браузере — pose estimation из видео, метрики (темп, амплитуда, симметрия, глубина) + rule-based проверки техники по упражнению. Никакой серверной части
